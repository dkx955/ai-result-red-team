What we look for

  • retrieve-vs-derive — did the fix come from the run's own reasoning, or from upstream lookup / git-history mining?
  • validation ↔ held-out gap — does it hold on data it couldn't have seen, or only on the visible proxy?
  • test / harness tampering — were tests skipped, deleted, weakened, or the harness relaxed to turn the light green?
  • honest scope — the one condition, stated, that would make the claim false.
